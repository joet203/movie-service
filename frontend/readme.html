  • MOVIES_DB_PATH: DuckDB file path (default movies.duckdb)
  • MOVIES_MAX_QUERY_LIMIT: max allowed query limit (default 50000)
  • MOVIES_TASK_POLL_INTERVAL_SECONDS: SSE polling interval (default 0.2)
  • Data persists across restarts because DuckDB is on-disk.
  • Use make run for clean demo starts (old DB files removed first).
  • Dashboard demo settings can inject artificial phase delays to force visible task progress.
MethodPathDescription

POST/datasetsUpload CSV (202 + task ID)

GET/moviesQuery rows (200 direct or 202 task handoff if >2s)
POST/movies/queryExplicit async query endpoint
GET/tasks/{id}/resultsFetch completed async query results
GET/datasets/downloadDownload gzip export (200 or 202 task handoff)
GET/tasks/{id}/downloadFetch completed async export artifact
GET/Dashboard UI
GET/sample-dataDownload sample CSV
ParamDescription
start_year, end_yearYear-range filters
genreCase-insensitive substring genre match
genres_allComma-separated AND match (all genres must be present)
sort_by, sort_orderSort field and direction
limit, offsetPagination controls

Responses include X-Total-Count; query responses also include X-Query-Time in extended version.

# Upload
curl -X POST -F "file=@movies.csv" http://localhost:8000/datasets

# Query with match-all genres
curl -v "http://localhost:8000/movies?start_year=2020&end_year=2023&genres_all=Action,Crime&sort_by=rating&sort_order=desc&limit=50"

# Async query flow
curl -X POST "http://localhost:8000/movies/query?genre=Action"
curl -N http://localhost:8000/tasks/TASK_ID/events
curl http://localhost:8000/tasks/TASK_ID/results

# Download
curl -o movies.csv.gz http://localhost:8000/datasets/download

# Slow download path
curl -N http://localhost:8000/tasks/TASK_ID/events
curl -o movies.csv.gz http://localhost:8000/tasks/TASK_ID/download
