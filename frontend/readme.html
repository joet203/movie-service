Build and Run

Quick reviewer guide for running the service and testing the required flows. For detailed rationale, open Design Choices.

Quick Start

uv sync
make run

UI: http://localhost:8000/ | API docs: http://localhost:8000/docs

Core Endpoints

MethodPathPurpose
POST/datasetsUpload CSV and start ingestion task
GET/moviesQuery by year range/genre (auto-task handoff if slow)
POST/movies/queryAlways async query endpoint
GET/datasets/downloadDownload gzip export (auto-task handoff if slow)
GET/tasks/{id}/eventsSSE progress stream
GET/tasks/{id}/resultsFetch completed async query rows
GET/tasks/{id}/downloadFetch completed async export artifact

Query Filters

  • start_year and end_year for year range
  • genre for substring filter
  • genres_all for match-all genres (AND), e.g. Action,Crime
  • sort_by, sort_order, limit, offset

Demo Tips

  • Use the Dashboard "Demo Settings" to force longer operations and observe SSE progress.
  • Data persists in movies.duckdb across restarts by default.
  • Use make run for a clean start (removes old DB files first).
